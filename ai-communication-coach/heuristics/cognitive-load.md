# Cognitive load

## Purpose

Fast rules for spending the user's attention, which is the only budget that matters.

## Principles

- The button communicates. So does the layout, the default, the empty space. Copy is one voice in a chorus — often not the loudest.
- Every element on screen bills the same attention account. Words compete with words, but also with everything else.

## Rules

- A screen carries about three messages before they blur into one.
- Twelve words or fewer scans in one pass.
- If the design says it, the copy doesn't have to.

## Decision heuristics

- Before adding a sentence, ask what element already says it. A disabled button says "not yet" better than a paragraph.
- If users keep missing a message, check what louder thing is outshouting it before making the message louder.

## Common mistakes

- Adding copy to compensate for a loud UI. Now there are two loud things.
- See `../mistakes/too-many-options.md`.

## Examples

- "Oops!" cut from the upload error: the error state itself already communicated that something went wrong.

## Counterexamples

- Awaiting one: a case where more copy genuinely lowered load.

## Exceptions

- Accessibility: information carried only by visuals must also exist in text for screen readers. "The design says it" has to include everyone.
