# Skipped by default

## Purpose

Some page elements users skip without deciding to. Protect their scarcity, or lose them entirely.

## Principles

- Users invest in these elements only when exceptionally interested. Often they won't realize the text is on the page — sometimes even when they're explicitly looking for it.
- Overuse programs the brain: every unnecessary instance trains users to ignore the element class everywhere, including the one time it matters.

## Rules

The skip list (not comprehensive):

1. Anything positioned above the first item in a list or chart (excluding the list or chart's controls).
2. Explanatory text, header text, introductory text.
3. Banners.
4. Descriptions.
5. Paragraphs.

- Never place a need-to-act message in a skip-zone element.
- Budget these elements like interruptions: each unnecessary one spends the attention the necessary one will need later.

## Decision heuristics

- Before adding explanatory text above a list, assume it is invisible. Plan as if nobody reads it — because mostly nobody does.
- If a message must be seen, attach it to the thing users came for: the list item, the field, the button — not the space above it.
- If an element from the skip list keeps growing, that's demand for a different surface, not for more text.

## Common mistakes

- An intro paragraph explaining a table that the table should explain itself.
- Stacked banners — each one halves the others.
- Putting the answer to a support-ticket question in a description, then wondering why the tickets continue.

## Examples

- Awaiting the first documented case (a skip-zone message users missed while actively hunting for it would be the perfect specimen).

## Counterexamples

- Exceptionally interested users do read these elements — that's who descriptions and paragraphs are *for*. The rule is scarcity, not abolition.

## Exceptions

- Legally required placements may force text into a skip zone. Put the user-critical version somewhere seen, too.
