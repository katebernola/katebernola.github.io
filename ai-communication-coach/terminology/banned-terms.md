# Banned terms

Words that don't ship. Each has a replacement — banning without an alternative just moves the argument.

*Canonical data: [`../rules.js`](../rules.js) → `INCLUSIVE_TERMS`, `COMPLEX_WORDS`, `VAGUE_PHRASES`, `IDIOMS` — edit rules.js first, then these tables.*

## Exclusionary

| Banned | Use instead |
|---|---|
| whitelist | allowlist |
| blacklist | blocklist |
| master / slave | primary / replica |
| crazy, insane | surprising |
| sanity check | quick check |
| guys | everyone |
| grandfathered | legacy |
| dummy | placeholder |
| blind spot | gap |

## Filler that hides the message

just · really · very · simply · actually · basically · quite · totally · please note · at this time (→ now) · we're excited to announce

## Vague phrases that name no cause

The specific problem is its own announcement — once the cause is named ("Files over 2GB won't upload"), a generic wrapper beside it is redundant and gets cut.

- "Something went wrong" — say what, if known; say what is known, if not.
- "An error occurred"
- "Invalid input"
- "Unable to complete"
- "There was a problem"

## Words that undermine or panic

- "Oops" — never; it undermines confidence. *(source: Intuit word list)*
- "Sorry" as filler — apologize only like a person, when we caused it. *(source: Polaris error messages)*
- "black box" → "unclear"; "dark patterns" → "deceptive design". *(source: Intuit word list)*

## Idioms that don't translate

oops · whoops · hang tight · heads up · hiccup · hit a snag · in a jiffy
