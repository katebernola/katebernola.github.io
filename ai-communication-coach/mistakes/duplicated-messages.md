# Duplicated messages

## What it looks like

The same news lands twice: on screen and in an email, in a banner and a notification, in two sentences of the same paragraph saying nearly the same thing.

## Why it happens

- Nobody owns the message, so every channel adds its own copy of it.
- Fear that users missed it the first time — solved by shouting instead of by placing it better.

## How to catch it

- Word-overlap between messages (the standards engine flags near-duplicates and on-screen/deferred overlap).
- Ask: if this message disappeared from one channel, would anything break?

## The fix

One channel per message. If the same news truly must land twice, stagger it deliberately and give both instances one owner.

The IA version of the rule: **one home, many doors**. Content lives in exactly one place; link to it from everywhere it's needed. Duplication isn't availability — it's future contradiction. *(source: Polaris information architecture)*
