# Task

## What it is

An actionable to-do assigned to a user inside the product — a message with a checkbox attached.

*Domain component. Rules below are starting points; refine from real cases.*

## Use it for

- Work the user must actually do, with a clear done-state.

## Don't use it for

- Announcements dressed as work. A task that can't be completed erodes trust in the whole task list.
- Education. A task teaches by being doable, not by explaining.

## Copy rules

- Title is verb-first: the action, then the object ("Review payroll for March").
- The title must make sense alone — task lists get scanned, not read.
- Include the deadline as a fact, not as urgency theater.

## Open questions

- Character budget — not yet measured against the real component.
- What happens to copy when a task expires uncompleted?
