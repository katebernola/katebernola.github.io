# Intelligent dashboard

## What it is

A surface that decides what to show each user — prioritized cards, tasks, and insights rather than a fixed layout.

*Domain component. Rules below are starting points; refine from real cases.*

## Use it for

- Surfacing the few things that matter now, in priority order.

## Don't use it for

- Everything. A dashboard that shows everything prioritizes nothing.

## Copy rules

- Each card is one atomic message with one verdict-worthy idea. The evaluation questions apply per card: does the user need this to act? Do they already know it?
- Position one gets read; everything after gets scanned. Rank cards like messages. See [Visual hierarchy](../heuristics/visual-hierarchy.md).
- A card the user can't act on is a report, not a message — question its place.

## Open questions

- How many cards before they blur? (Screens carry about three messages; does the rule transfer?)
- Who owns the copy of machine-selected cards?
