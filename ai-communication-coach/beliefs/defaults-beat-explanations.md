# Defaults beat explanations

## Belief

A good default does more than a good explanation. Users accept a sensible pre-made choice far more often than they absorb the reasoning needed to make it themselves.

## Evidence

- Awaiting a documented case. Currently held on general design experience, not on recorded evidence from this system's own patterns — which is exactly why confidence is only medium.

## The condition

A default only beats an explanation when it can defend itself: the label states the verifiable fact that earned it the default ("most common," "cheapest"). An unexplained or superlative-labeled default ("Best value") stops being a service and starts being a sale. See [Indefensible defaults](../mistakes/indefensible-defaults.md).

## Exceptions

- Consequential, irreversible, or costly choices: a default that decides for the user shifts responsibility they didn't agree to carry.
- Anywhere regulation requires an explicit informed choice.

## Confidence

Medium — promote or demote when the first real case lands.
