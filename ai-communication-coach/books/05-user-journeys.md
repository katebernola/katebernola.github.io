# 05 · User journeys

## Purpose

The six journey stages and what each demands from copy. The same words mean different things to a newcomer and a regular.

## Principles

- The reader's moment shapes the reading. Stage is decided before goals, because goals only make sense inside a moment.
- A stressed reader scans for "am I okay?" and "what do I do?" — in that order.

## Rules

| Stage | The user | What the copy must do |
|---|---|---|
| First contact | Just arrived; barely knows you | Say the one thing that helps the next step. New users are deciding whether to trust you, not reading. |
| Getting set up | Said yes; wants the good part | Attach each message to an action they're already taking. Tips read before they matter are tips forgotten. |
| Everyday use | Runs on habit | Only interrupt for things that change what they'd do next. Loud changes spend the calm you built. |
| Moment of need | Stuck, worried, or blocked | Answer "am I okay?" then "what do I do?" Everything else waits or goes. Terse beats warm-and-wordy here. |
| Decision point | About to commit | Give the deciding fact, next to the choice. Missing info doesn't stop people — it makes them guess or leave. |
| Leaving or lapsing | Already decided something | Be honest and quick. Copy that argues feels like a trap, and traps get remembered. |

*Canonical data: [`../rules.js`](../rules.js) → `STAGES`, `STAGE_CARDS`, `STAGE_MISMATCH`.*

## Decision heuristics

- Announcing features to someone mid-problem reads as noise, or as being ignored.
- A confirmation inside a habitual flow gets clicked on muscle memory — state the consequence, not "are you sure?"
- An error at first contact may be the only thing that user ever reads from you.

## Common mistakes

- Choosing the stage by which screen it is, instead of by what the user is going through on it.
- See `../mistakes/marketing-in-product.md` for stage-blind upsells.

## Examples

- Upload error at moment of need: reassure first ("your other files are safe" earned support placement), instruct second.

## Counterexamples

- Awaiting a real case where a mid-task interruption improved outcomes.

## Exceptions

- Legally required notices ignore stage. Place and rank them deliberately anyway.
