Clarifying in-app messaging governance

Eliminating redundant comms by fixing the intake process

The problem

Product owners submitted in-app messaging requests to both marketing and UX independently. Each group assessed requests using different criteria. UX had no authority to decline messaging, so the same information landed in multiple channels without coordination. Clients complained: "You tell me so many things in so many places." Exec escalations were frequent because UX was trapped in a no-saying role.

Before: parallel intake, conflicting assessments

  • PO submitted the same request to marketing and UX in parallel
  • Each group ran its own discovery, using its own criteria
  • UX had no framework to decline messaging; assessments often conflicted
  • Same message could appear across multiple channels uncoordinated
  • Clients saw redundant information
  • Escalations climbed because UX was forced into a gatekeeping role without authority

Parallel paths, separate criteria, conflicting outcomes.

After: single intake, shared decision framework

  • PO submits one request through marketing intake
  • Marketing evaluates channel strategy across all surfaces
  • Marketing determines if the message is a candidate for in-app via shared criteria
  • UX consults only when the message meets criteria and needs placement guidance
  • One unified comms plan; UX never has to decline
  • Escalations drop to match marketing's historical rate

Single intake, gate-keeping at the criteria level, unified delivery.

In-app messaging criteria

A message is appropriate in-app only when all of the following are true:

  1. It impacts at least 5,000 clients or 10,000 users
  2. We know exactly what screen the user is on when they need the information
  3. The user is likely to stop their current task to address it immediately
  4. Value (time or money saved × impacted clients) exceeds disruption cost (disruption per client × total viewers)

If criteria 1 is "no" or criteria 2 is "we don't know," the message is not a candidate for in-app. UX is not engaged.

UX role clarity

  • Evaluates placement, information architecture, and user impact
  • Does not evaluate channel strategy or prioritization
  • Created a Figma kit for consistent in-app treatments
  • Applied existing placement strategy from change management work
  • Brought long-standing experience assessing comms needs for customer-facing changes

Cross-functional alignment

  • Co-created decision framework with marketing comms and product ops
  • Marketing adopted criteria quickly via learn-by-doing
  • Intake process required only one additional checkbox
  • PO now facilitates one discovery conversation instead of two
  • PO receives one unified answer from one team instead of parallel assessments

Impact

  • Eliminated redundant messaging across channels
  • Addressed the client pain point: "You tell me so many things in so many places"
  • Removed UX's gatekeeping role without authority
  • Dropped exec escalations to match marketing's historical rate
  • Improved UX efficiency by removing unnecessary intake and debate cycles
  • Strengthened governance without adding process overhead
  • Increased consistency and clarity across all client communications
